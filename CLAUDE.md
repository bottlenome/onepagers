# CLAUDE.md

このリポジトリでの開発ルール。

## ダッシュボード更新ルール

プロジェクトのコードを変更した場合、関連するダッシュボードも必ず更新すること。

1. **プロジェクト個別の `dashboard.md`** — 機能追加・変更・バグ修正時に該当プロジェクトの `<project>/dashboard.md` を更新する
2. **ルートの `dashboard.md`** — プロジェクトの追加・削除・説明変更時にプロジェクト一覧テーブルを更新する
3. **ルートの `index.html`** — プロジェクトの追加・削除・説明変更時にカードの内容を更新する

### チェックポイント

- 新規プロジェクト追加時: `<project>/index.html`, `<project>/dashboard.md`, ルート `dashboard.md`, ルート `index.html` の4箇所を更新
- 既存プロジェクト変更時: `<project>/dashboard.md` の機能リストや説明が現状と一致しているか確認
- 説明文の変更時: ルート `dashboard.md` のテーブルとルート `index.html` のカードの両方を同期

## ゲーム開発の設計指針（櫻井政博メソッド）

ゲームの新規作成・改修時は `sakurai-methodology/index.html` に記載された設計原則を参照すること。
以下は特に重要な原則の要約。

### コア原則

1. **リスクとリターン** — ゲーム性＝かけひき＝「リスクを抑えてリターンを得る工夫」。全てのゲームメカニクスでリスクとリターンのバランスを意識する
2. **カービィズム（初心者ファースト）** — 誰でも遊べて上級者も極められる両立設計。操作は直感的に、スキルシーリングは高く
3. **プレイヤー視点** — 開発者の都合ではなくプレイヤーが何を感じるかから設計を始める
4. **プレゼンはスピード** — 無駄をそぎ落とし必要なことだけ伝える。UI・チュートリアル・体験全体に適用
5. **遊びの仕事は遊びじゃない** — プロとして需要に応える

### 設計チェックリスト

ゲーム機能を追加・変更する際に確認すべき項目:

- [ ] **リスクとリターン**: この機能にかけひきの要素があるか？ ハイリスク行動にはハイリターンが対応しているか？
- [ ] **ギュッとしてパッと解消**: 緊張と解放のサイクルが体験として成立しているか？
- [ ] **目の前に吊られたごほうび**: プレイヤーの目の前に常に次のゴールが見えているか？
- [ ] **最初にクライマックスを**: 序盤で最高の体験を提供してプレイヤーを掴めているか？
- [ ] **平均と平凡は同じ**: キャラクター・職業・装備に個性的な尖りがあるか？
- [ ] **情報の優先順位**: 今プレイヤーが必要としている情報だけを見せているか？
- [ ] **最短導線**: 目的の操作に到達するまでのステップ数は最小か？
