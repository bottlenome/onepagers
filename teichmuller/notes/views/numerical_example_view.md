# 具体的な数値で理解する「足し算と掛け算の分離」

## この文書の目的

IUT理論の核心操作——「掛け算の次元だけを動かし、足し算の次元は固定する」——を
**実際の数値**で見せる。抽象的な定義ではなく、手で計算できるレベルに落とす。

---

## 準備: p進の世界での「2つの次元」を数値で見る

### 5進数の世界 (p=5)

5進整数とは 5 を「小さい数」と見なす世界。

```
通常の感覚: 125 は「大きい」、1/5 は「小さい」
5進の感覚: 125 = 5³ は「超小さい」、1/5 は「大きい」
```

5進付値 v₅(x) = 「x を何回 5 で割れるか」:

```
v₅(1)   = 0    （5で割れない）
v₅(5)   = 1    （1回割れる）
v₅(25)  = 2    （2回割れる）
v₅(125) = 3    （3回割れる）
v₅(7)   = 0    （5で割れない）
v₅(50)  = 2    （50 = 2 × 5²）
```

### 2つの次元の正体

任意の 5 進数 x ≠ 0 は**一意に**次のように分解できる:

```
x = 5^n × u     （n = v₅(x), u は5進単数）
    ~~~~   ~~~
    値群   単数群
    (掛け算の次元)  (足し算の次元)
```

| 数 x | 値群の成分 5^n | 単数群の成分 u | 分解 |
|------|--------------|--------------|------|
| 7 | 5⁰ = 1 | 7 | 7 = 1 × 7 |
| 25 | 5² | 1 | 25 = 5² × 1 |
| 50 | 5² | 2 | 50 = 5² × 2 |
| 35 | 5¹ | 7 | 35 = 5¹ × 7 |
| 375 | 5³ | 3 | 375 = 5³ × 3 |

**値群 = 「5を何回含むか」= 掛け算的な情報**
**単数群 = 「5を全部取り除いた残り」= 足し算に関わる情報**

### なぜ「単数群 = 足し算的」なのか

直感的でないので具体例で示す。

```
7 + 18 = 25

左辺: 7 = 5⁰ × 7,  18 = 5⁰ × 18    （どちらも値群=0、つまり単数）
右辺: 25 = 5² × 1                     （値群=2、単数=1）
```

**2つの単数（7と18）を足しただけなのに、値群が0から2に跳ね上がった！**

これが足し算と掛け算の「絡み合い」の最も単純な例:
- 掛け算的には 7 も 18 も「5とは無関係」（値群=0）
- でも足し算すると 7+18=25=5² で「5が2つ出てくる」
- **足し算が掛け算の構造を変えてしまう**

逆もある:
```
3 + 122 = 125 = 5³    （単数同士の和が値群=3を生み出す）
2 + 123 = 125 = 5³    （同上）
4 + 121 = 125 = 5³    （同上）
```

**環構造（足し算+掛け算が共存）がある限り、この「絡み合い」は避けられない。
これがABC予想を直接証明できない根本的な理由。**

## テート曲線のq-パラメータを数値で

### 設定

楕円曲線 E が素数 p=5 で乗法的悪い還元を持つとする。
このとき、5進体 Q₅ 上で E はテート曲線:

```
E ≅ G_m / q^Z     （q = 5進数で |q|₅ < 1）
```

具体的に **q = 5²** （つまり v₅(q) = 2）とする。

### テータ関数の値（l=11 の場合、l*=5）

各等分点 j = 1, 2, 3, 4, 5 でのテータ値は（主要項として）:

```
Θ(j) ∼ q^{j²}
```

| j | j² | q^{j²} = (5²)^{j²} = 5^{2j²} | v₅(q^{j²}) = 2j² |
|---|----|----|------|
| 1 | 1 | 5² = 25 | 2 |
| 2 | 4 | 5⁸ = 390,625 | 8 |
| 3 | 9 | 5¹⁸ ≈ 3.8 × 10¹² | 18 |
| 4 | 16 | 5³² ≈ 2.3 × 10²² | 32 |
| 5 | 25 | 5⁵⁰ ≈ 8.9 × 10³⁴ | 50 |

テータ値の付値の列: **2, 8, 18, 32, 50**

q-パラメータ自体の付値: **2**

## Θ-リンクが「やること」を数値で見る

### 2つの世界を用意する

```
┌──── 世界A（ドメイン側のホッジシアター）────┐
│                                            │
│  q_A = 5²     (v₅ = 2)                    │
│  テータ値: 5², 5⁸, 5¹⁸, 5³², 5⁵⁰          │
│  足し算: 5² + 5⁸ = 25 + 390625 = 390650   │
│         = 2 × 5² × 7813                   │
│  （足し算の結果は世界Aの環構造で決まる）     │
│                                            │
└────────────────────────────────────────────┘

          ┃ Θ-リンク: テータ値 ↔ q-パラメータ
          ┃
          ┃ 繋がるもの: 値群（v₅の値）
          ┃ 繋がらないもの: 単数群、足し算
          ▼

┌──── 世界B（コドメイン側のホッジシアター）───┐
│                                            │
│  q_B は世界Bの q-パラメータ                 │
│  Θ-リンクにより:                            │
│    v₅(q_B) は q_A のテータ値に関係する      │
│                                            │
│  しかし:                                    │
│    世界Bの足し算 ≠ 世界Aの足し算            │
│    世界Bの単数群 ≠ 世界Aの単数群            │
│    「390650」という数は世界Bに存在しない     │
│                                            │
└────────────────────────────────────────────┘
```

### 何が「動いた」か、何が「固定」か

具体的に見てみましょう。

**掛け算の次元（値群）: 動いた**

```
世界A:  q_A の付値 = 2
        テータ値列の付値 = {2, 8, 18, 32, 50}

世界B:  q_B の付値 = ???
        Θ-リンクにより、テータ値列 {2, 8, 18, 32, 50} と
        q_B の付値 2 が「同一視」される

        つまり: {2j²}_{j=1,...,5} が一つの「2」に対応
```

**これが「値群の伸縮」: 付値 2 のものが、2j² (= 2,8,18,32,50) に「広がった」列と対応づけられる。**

**足し算の次元（単数群）: 動かない（というか、切断されている）**

```
世界A: 7 + 18 = 25（5進で v₅ が 0→2 に跳ぶ。単数の足し算の結果。）

世界B: この「7 + 18 = 25」は世界Bでは意味をなさない。
       世界Bの「7」と世界Aの「7」は、Θ-リンクで繋がっていない。
       世界Bには世界B独自の足し算がある。
```

**通常の環構造では**:
- 値群と単数群は分配法則で結合している
- 例: 5² × 7 + 5² × 18 = 5² × (7+18) = 5² × 25 = 5⁴
- 値群の情報（5²）と単数群の情報（7, 18）が混ざり合う

**Θ-リンクでは**:
- 値群の情報だけを「パイプ」で別の世界に送る
- 単数群の情報は送れない → 分配法則が成り立たない → 環構造が壊れる

## 「歪み」が不等式を生むメカニズム（数値で）

### 対数体積で見る

「対数体積」= 付値の（正規化された）合計、と思ってください。

**世界Aのテータ値列の対数体積**:

```
Σ v₅(q^{j²}) = v₅(q) × Σ j²
             = 2 × (1 + 4 + 9 + 16 + 25)
             = 2 × 55
             = 110
```

**世界Bの q-パラメータの対数体積**:

```
v₅(q_B) = 2
```

**Θ-リンクによる「歪み率」**:

```
110 / 2 = 55 = Σ j² (j=1,...,5)
```

### これがABC予想にどう繋がるか

一般の l（l* = (l-1)/2）では:

```
歪み率 = Σ_{j=1}^{l*} j² ≈ l³/24
```

この歪み率を l で割って正規化すると:

```
(1/l) × Σ j² ≈ l²/24
```

さらに、テータ値列は l* 個の成分からなるので、1成分あたりの平均付値は:

```
(Σ j²) / l* ≈ l²/6
```

一方、q-パラメータの対数は楕円曲線の高さ ht_E に対応する。

IUT IVで l を ht_E と同じオーダーに選ぶと（l ∼ ht_E）、上の見積もりから:

```
(1/6) × ht_E ≤ (1+ε) × (log-diff + log-cond) + 定数
```

の形の不等式が導かれる。
左辺は (1/6) × log(q) に対応し、右辺は判別式と導手に対応する。

### 具体的な数字の流れ

q = 5², l = 11 (l* = 5) の場合:

```
1. テータ値列: {5², 5⁸, 5¹⁸, 5³², 5⁵⁰}
   付値列: {2, 8, 18, 32, 50}

2. 対数体積（合計）: 110

3. q-パラメータの対数体積: 2

4. 比率: 110/2 = 55

5. 1/l で正規化: 55/11 = 5 = l*

6. 1/l* で平均: 55/5 = 11 = l

7. この「11」が最終的に (1/6)·ht_E の左辺になる
   （厳密にはもう少し複雑だが、最高次項として）
```

## なぜ「分離」が不等式を生むのか（核心の直感）

通常の環構造の中では:

```
足し算と掛け算が絡み合っている
  → 値群の変化（q → q^{j²}）を見積もろうとすると、
    足し算由来の項（ガウス極）が爆発する
  → 不等式が vacuous（空虚）になる（何も言えない）
```

IUT理論では:

```
足し算の次元と掛け算の次元を分離する
  → 掛け算の次元（値群）だけを q → q^{j²} で変形
  → 足し算の次元は「対数殻」の中に閉じ込められる
  → 対数殻のサイズは有限（明示的に計算可能）
  → 歪みの量 = Σj² / q の比率が有限に制御される
  → 非自明な不等式が得られる
```

**分離しないと**: ガウス極のせいで「∞ ≤ 何か」になり、何も言えない。
**分離すると**: ガウス極が消えて「有限の量 ≤ 何か」になり、ABC予想の形になる。

## 古典的タイヒミューラーとの数値的対応

### 古典: z → ζ = Kx + iy

```
リーマン面のタイヒミューラー変形:

z = x + iy = (3, 4)    （2つの実次元）

変形後: ζ = 2x + iy = (6, 4)     （K=2 で x方向だけ2倍）

動いた: x成分  3 → 6 (×2)
固定:   y成分  4 → 4 (そのまま)

歪みの量 = K = 2（擬等角定数）
```

### IUT: 値群を j² 倍に「伸縮」

```
数体のIUT変形:

q = 5²   （v₅ = 2, 単数 = 1 の2つの次元）

変形後 (j=3): q^{j²} = q^9 = 5¹⁸

動いた: 値群  2 → 18 (×9)
固定:   単数群は別世界に属するので比較不能（= 固定というより切断）

歪みの量 = Σj² = 55（l*=5の場合）
```

| | 古典 | IUT |
|---|---|---|
| 対象 | z = x + iy | q = p^n × u |
| 2つの次元 | 実部 x / 虚部 y | 値群 n / 単数群 u |
| 変形 | x → Kx | n → j²n |
| 固定 | y → y | u → （別世界、繋がない）|
| 歪みの量 | K（擬等角定数）| Σj²（テータ値の合計）|
| 結論 | テータ距離の上界 | ABC不等式 |
