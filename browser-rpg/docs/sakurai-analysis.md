# Loveless Chronicle 櫻井メソッド分析

櫻井政博のゲームデザイン方法論 (`sakurai-methodology/index.html`) を基準に Loveless Chronicle を分析した結果。

## 総合評価

| 観点 | 評価 | 課題 |
|------|------|------|
| リスクとリターン | B- | クラフトのリスクは良い。戦闘死亡のペナルティが軽すぎる。序盤にかけひきがない |
| ギュッとしてパッと解消 | C+ | 自動戦闘のため緊張は「観察」であり「参加」ではない |
| 最初にクライマックスを | F | スライム戦から始まる。フックがない |
| チュートリアル | D | 隠しパラメータの説明なし。転職ループの目的が不明 |
| UI情報設計 | B | 清潔で機能的だが成長率が非表示。装備比較がない |
| 平均と平凡は同じ | B+/D | 上級職は個性的。下級職はスキルなしで体感が同一 |
| 目の前に吊られたごほうび | C | レシピは見える。ボス・ドロップ・全体構造が見えない |
| 初心者ファースト | C+ | 表面は簡単だが、深い仕組みが隠れすぎて浅く見える |

## 改善提案

### 優先度: 高

#### 1. 最初にクライマックスを — オープニング体験の刷新

**現状:** 名前入力 → 職業選択 → 始まりの町 → 草原 → スライム戦。

**問題:** ゲームの最も面白い瞬間（ボス戦・上級職スキル・レアドロップ）に到達するまでに長い退屈な道のりがある。DEBUG_PRESETにLv30騎士+伝説の剣が設定されており、開発者自身がそこが面白いと知っている。

**提案:** ゲーム開始時にLv30プリセットで「ドラゴン戦」のデモバトルを見せる。勝っても負けてもフラッシュフォワードとして機能し、「このゲームはここに向かうんだ」と伝える。その後「X年前…」として通常スタート。

#### 2. 転職ループの可視化 — コアループを「ごほうび」に変える

**現状:** ゲームの中核メカニクス（転職ループによるステータス蓄積）がプレイヤーに一切説明されない。ボスは2〜20ループを前提に設計されているが、プレイヤーはそれを知らない。

**問題:** Lv30に到達してドラゴンに負けたプレイヤーは「何をすればいいかわからない」状態になる。

**提案:**
- ステータス画面に「転職回数」「累計成長値」を表示する
- 転職場に「転職するとLv1に戻るが、成長の50%を引き継ぐ。ループするほど強くなる」という説明を追加
- ボスに推奨ループ数のヒントを追加（例: 「このボスは2回以上転職した冒険者が挑むべき」）

#### 3. 成長率の可視化 — 隠しパラメータを「見せるかけひき」に変える

**現状:** 成長率（職業の `rate` + 装備の `g`）は完全に隠されている。装備が成長率に影響することをプレイヤーは知りえない。

**問題:** ゲーム最深部の最適化パズル（どの装備でどの職業を育てるか）が存在しないも同然。

**提案:**
- ステータス画面に各パラメータの成長率をバー表示する（「高」「中」「低」程度でよい）
- 装備に成長率ボーナスの表記を追加する（例: 「攻撃成長 +8%」）
- レベルアップ時に「成長率のおかげで追加ボーナス！」と表示する

### 優先度: 中

#### 4. 下級職の差別化 — 「平均と平凡は同じ」への対処

**現状:** 下級職4種はスキルがなく、ステータス傾向が違うだけ。戦闘中の体験は全て「通常攻撃」で同一。特に僧侶は「バランス型」＝何も尖っていない。

**提案:**
- 各下級職に1つだけ固有パッシブを付与する（戦士: 被ダメ-10%、魔法使い: MP自然回復、僧侶: 戦闘後HP10%回復、盗賊: ドロップ率+5%）
- これだけで「どの職業で育てるか」のかけひきが生まれる

#### 5. 戦闘死亡のリスク追加

**現状:** 死亡時は最後の町にHP1で戻るだけ。ゴールドもEXPもロスなし。

**問題:** 「リスクとリターン」のリスク面がない。高層に挑む動機はあっても、失敗のコストがないためかけひきにならない。

**提案:** 死亡時にゴールドの10-20%を失う（銀行に預けたゴールドは安全）。銀行システムが既に存在するので、「銀行に預けるか、手持ちで持つか」というかけひきが自然に生まれる。

#### 6. ボスと進行構造の可視化

**現状:** ボスの存在は遭遇するまで不明。ドロップ表も非公開。

**提案:**
- 冒険者ギルド的な施設を始まりの町に追加し、「この世界の脅威」としてボス一覧（シルエット + 推奨レベル + 既知の弱点）を表示
- 図鑑システム: 倒したモンスターのドロップ情報を記録する

### 優先度: 低

#### 7. 装備比較UI

**現状:** ショップで装備を見ても、現在装備中のアイテムとの比較ができない。

**提案:** 購入画面で「現在: 攻撃+5 → 購入後: 攻撃+10 (+5)」のような差分表示を追加。

#### 8. ショートカット「最後に探索した場所へ」

**現状:** 町から冒険に出るたびに階層を1つずつ進む必要がある。

**提案:** フィールド画面に「最深到達階層へワープ」ボタンを追加（1クリック短縮）。

## 櫻井メソッド適用のまとめ

このゲームの設計上の最大の問題は「**コアループ（転職サイクルによるステータス蓄積）が完全に不可視**」であること。ゲームの最も面白いシステム（隠し成長率 × 職業 × 装備の最適化パズル）がプレイヤーに届いていない。

櫻井の言葉を借りれば:
- 「目の前に吊られたごほうび」がない → 転職ループのゴールが見えない
- 「お客さんに説明できるの？」→ 隠しパラメータを説明できない
- 「最初にクライマックスを」→ 最高の瞬間が最後に来る
- 「平均と平凡は同じ」→ 下級職が没個性

逆に良くできている点:
- 上級職5種の個性設計は「平均と平凡は同じ」をクリアしている
- 強化システムの破壊リスクは「リスクとリターン」が機能している
- 4つの作戦プリセットは「カービィズム」的な簡易操作設計
- 町のUI配置は「最短導線」を意識した1クリックハブ設計
