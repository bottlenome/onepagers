## 3つの不確定性を数値で見る

IUT III 定理B / 系3.12では、マルチラジアル表現を構成する際に
避けられない3つの不確定性が同定される。

### (Ind1): ガロア群の内部自己同型の不確定性

**由来**: 遠アーベル幾何学的復元で、エタール基本群 Π からスキームを復元するとき、
Π の内部自己同型（= 元による共役操作）による不定性が残る。

```
復元: Π → X → Π_X → X'
                ↑
           Inn(Π) が作用  ← これが Ind1
```

**数値的な影響**:

Ind1は「ラベルの置換」として現れる。プロセッションの枠組みでは、
ラベル集合 S±_{j+1} = {0, 1, ..., j} の任意の置換自己同型として作用する。

```
j=3 のとき:  S±₄ = {0, 1, 2, 3}

Ind1 の効果:  {0,1,2,3} の置換 = 24通り

ラベル 3 に対応するテータ値 q^{9} の「位置」が 4通りに不確定になる
（置換で 3 が送られ得る先が {0,1,2,3} の4つ）
```

プロセッションを使う理由がここにある: 一度に全ラベルの置換を考えると不確定性が大きすぎるが、
段階的に考えれば各段階で (j+1) 通りに抑えられる。

### (Ind2): 単数群 O^× の作用（Z^× 不確定性）

**由来**: Θ-リンクを跨ぐ際、F⊢×μ-プライムストリップの自己同型として
Z^× が O^×μ に作用する。

**数値的な影響**:

O₅^×μ = O₅^× / (5進の1の冪根) への Z₅^× の作用。

```
テータ値 q^{j²} = 5^{2j²} の「単数部分」に Z₅^× が作用

具体例:
  テータ値 5² の「単数部分」1 に対して:
    Z₅^× ∋ α が作用 → 1 · α = α ∈ O₅^×μ

  つまり、テータ値を「付値 + 単数」に分解したとき:
    付値部分（= 2j²）: 変わらない（Ind2 に対して不変）
    単数部分: Z₅^× の軌道全体に不確定化
```

重要: テータ関数の**定数倍剛性**（[EtTh, Cor 2.19]）により、
この不確定性は μ_{2l}（1の2l乗根群）のみ。つまり:

```
Ind2 による不確定性は有限群 μ_{22}（1の22乗根）の作用にすぎない
→ 対数体積への影響は log(22) 程度の定数
→ 最終不等式では「定数項」に吸収される
```

さらに、mono-theta環境の理論の核心的な帰結として:

```
μ_{22} → O₅^× → O₅^×μ

この合成は自明写像（μ₂₂ の像は O₅^×μ で消える）
→ Z^× 不確定性から「隔離される」（[IUT III, Fig. I.3]）
```

### (Ind3): 対数殻の上半可換性による不確定性

**由来**: log-Kummer対応において、Kummer同型が log-リンクと
「元のレベルでは」整合しない。対数殻という「少し大きい入れ物」を使うことで
初めて整合性が得られるが、その分だけ上界に余裕（= 不確定性）が出る。

**数値的な影響**:

```
log-リンクのドメイン側: O₅^▷ ⊆ I₅  （整数環が対数殻に含まれる）
log-リンクのコドメイン側: log₅(O₅^×) ⊆ I₅  （対数の像も含まれる）

両方含む入れ物 = I₅
→ 「正確な等式」ではなく「上界」しか得られない
→ 対数体積の計算が等式ではなく不等式になる原因
```

具体的には、O₅^▷ の対数体積（= 0）と I₅ の対数体積の差分が
(Ind3) の「コスト」:

```
μ_log(I₅) - μ_log(O₅^▷) = log(p) = log(5) ≈ 1.61

5進の1素点あたりのコスト ≈ 1.61
```

## マルチラジアル表現の数値的な意味

### マルチラジアルとは

マルチラジアルアルゴリズム = Θ-リンクの**両側**から同じ出力を与えるアルゴリズム。

```
†HT（ドメイン側）から見たテータ値   →  アルゴリズム A  →  出力 X
‡HT（コドメイン側）から見た q-パラメータ →  アルゴリズム A  →  出力 X'

マルチラジアル ⟺ X = X'（適切な意味で）
```

### 数値で見るマルチラジアル表現

p=5, q=5², j=3 の場合:

```
ドメイン側（†HT）:
  テータ値 = q^{j²} = 5^{18}
  これを対数殻 I₅ の中で表現する:
    v₅(5^{18}) = 18 → 「付値18」のラベルが付いたI₅の元

コドメイン側（‡HT）:
  q_B = 5² → v₅ = 2 → 「付値2」のラベルが付いたI₅の元

マルチラジアル表現:
  両側のデータを「対数殻のテンソルパケットの中の元」として統一的に記述
  → (Ind1) + (Ind2) + (Ind3) の不確定性を込みで
```

### Θ-パイロットと q-パイロット

IUT理論の用語で:

- **Θ-パイロット** = Θ-リンクのドメイン側のテータ値列 {q^{j²}}
- **q-パイロット** = Θ-リンクのコドメイン側の q-パラメータ

マルチラジアル表現は、Θ-パイロットをコドメイン側の座標系で記述する。
この記述には不確定性が伴うが、対数殻が「共通の容器」として機能する:

```
Θ-パイロット {5², 5⁸, 5¹⁸, 5³², 5⁵⁰}
  ↓ マルチラジアル表現（Ind1, Ind2, Ind3 込み）
コドメイン側の対数殻のテンソルパケットの中に着地

q-パイロット 5²
  ↓ そのまま
コドメイン側の対数殻の中に着地

両者の対数体積を比較 → 不等式が得られる
```

## 対数体積の計算 — IUT IV 定理1.10の数値追跡

### 計算の全体構造

IUT IV §1 では、IUT III 系3.12 の抽象的な見積もりを
具体的な数論的不変量の関数として表現する。

計算の流れ（[IUT IV, Theorem 1.10]）:

```
Step 1: 各非アルキメデス付値 v での対数殻のテンソル積の対数体積を計算
Step 2: アルキメデス付値での対数体積を計算
Step 3: 素数分布の古典的定理を適用
Step 4: 重み付き平均の評価
Step 5: 全部を組み合わせて明示的な不等式を得る
```

### Step 1: 非アルキメデス対数体積（命題 1.1–1.4）

p=5 での計算。テンソルパケットの対数体積を見積もる。

テンソルパケット = 対数殻のテンソル積:

```
プロセッションの各段階 S±_{j+1} = {0, 1, ..., j} に対し:
  テンソルパケット_{j} = ⊗_{i ∈ S±_{j+1}} I₅

対数体積: μ_log(テンソルパケット_{j}) = (j+1) × μ_log(I₅)
```

「Θ-パイロットの可能な像の全体」がこのテンソルパケットに含まれる
（[IUT IV, proof of Thm 1.10, Step (v)]）:

```
Θ-パイロットの像 ⊆ テンソルパケット_{j} の正規化 (R_I)^∼

Ind1 と Ind2: 自己同型 φ の任意性で吸収
Ind3: 上界を取ることで吸収
```

p=5, j=3 の場合の具体的な対数体積:

```
テンソルパケット₃ = I₅ ⊗ I₅ ⊗ I₅ ⊗ I₅  （4つのテンソル）

判別式の寄与: d_{K_v} = v₅(different of K/Q₅)
  Q₅ 上では different = 1 なので d = 0

テンソルパケットの対数体積 ≤ 4 × μ_log(I₅)
  = 4 × log(5)
  ≈ 4 × 1.61
  = 6.44
```

### Step 2–3: プロセッション正規化上界

j = 1, ..., l* にわたる平均を取る。
IUT IV の証明の Step (v) で得られる「プロセッション正規化上界」:

```
(l*+3)/2 × log(d_{K_{vQ}}) − (2l*+1)(l*+1)/(12l) × log(q_{vQ}) + ...
```

p=5, l=11, l*=5 の数値:

```
係数:
  (l*+3)/2 = (5+3)/2 = 4
  (2l*+1)(l*+1)/(12l) = (11)(6)/(132) = 66/132 = 1/2

プロセッション正規化上界 ≤ 4·log(d) − (1/2)·log(q) + (誤差項)
```

さらに簡略化すると（[IUT IV, Theorem 1.10] の最終行）:

```
(l+1)/4 × [(1 + 4/l)·log(d) − (1/6)·log(q) + 小さい項]

l=11:
  (l+1)/4 = 3
  1 + 4/l ≈ 1.36
  1/6 ≈ 0.167

上界 ≤ 3 × [1.36·log(d) − 0.167·log(q) + 小さい項]
```

### Step 4–5: 最終不等式の導出

IUT III 系3.12 の不等式 C_Θ ≥ -1 を適用。
[IUT IV, Theorem 1.10] の結論:

```
(1/6)·log(q) ≤ (1 + 20·d_mod/l)·(log(d_{F_tpd}) + log(f_{F_tpd})) + 20·(e*_mod·l + η_prm)
```

各項の意味を数値で:

```
左辺: (1/6)·log(q)
  = q-パラメータの対数の 1/6
  = 楕円曲線の「高さ」に比例

右辺第1項: (1 + 20·d_mod/l)·(log(d) + log(f))
  ≈ (1 + ε)·(判別式 + 導手)
  l を大きく取ると 20·d_mod/l → 0 なので (1+ε) の ε を任意に小さくできる

右辺第2項: 20·(e*_mod·l + η_prm)
  = 定数項（l と基本的不変量に依存するが、楕円曲線の高さには依存しない）
```

## 数値例: (1/6)·log(q) の意味

### 1/6 はどこから来るか

IUT IV の証明の Step (v) で:

```
プロセッション平均:  (1/n) × Σ_{m=1}^{n} m² = (2n+1)(n+1)/6

n = l* = (l-1)/2 のとき:
  (1/l*) × Σ_{j=1}^{l*} j² = (2l*+1)(l*+1)/6 = l(l+1)/[6(l-1)]

l → ∞ のとき → l/6 → 1/6 × l
```

つまり:

```
歪み率の平均 = Σj² / l* ≈ l²/6
これを l 自体で割ると ≈ l/6
さらに l ∼ ht_E（高さ）のオーダーに選ぶと
(1/6)·ht_E が左辺に出現
```

### l=11 での具体的な値

```
Σ_{j=1}^{5} j² = 1+4+9+16+25 = 55

55 / l* = 55/5 = 11 = l

55 / l = 55/11 = 5 = l*

(2l*+1)(l*+1)/6 = 11·6/6 = 11 = l    ← (E2) の帰結
```

これが「l/6 の項」の正体: テータ値列 {q^{j²}} の j² の平均が、
プロセッションの正規化を経て l/6 のオーダーに集約される。

## 最終不等式からABC予想へ

### IUT IV 定理A (= Corollary 2.3)

定理1.10の楕円曲線に特化した結果を、[GenEll] の理論で一般化:

```
ht_{ω_X(D)} ≤ (1+ε)·(log-diff_X + log-cond_D)
```

### ABC予想への特殊化

X = P¹, D = {0, 1, ∞}（射影直線から3点を除く）の場合:

a + b = c, gcd(a,b) = 1 に対応する点 P = [a:c] ∈ P¹\{0,1,∞} を考える:

```
ht_{ω_X(D)}(P) ∼ log(max(|a|,|b|,|c|)) = log(c)

log-cond_D(P) ∼ log(rad(abc))

log-diff_X(P) ∼ 定数（有限個の例外を除く）
```

したがって:

```
log(c) ≤ (1+ε)·log(rad(abc)) + C(ε)

⟺  c ≤ K(ε) · rad(abc)^{1+ε}
```

これがABC予想。

### 具体的な数値例

a = 2, b = 3¹⁰ - 2 = 59047, c = 3¹⁰ = 59049 の場合:

```
c = 59049 = 3¹⁰
rad(abc) = rad(2 × 59047 × 3¹⁰) = rad(2 × 59047 × 3)

59047 = 59047（素数かチェック: 実は 59047 = 7 × 8435.28... ではない、
59047 は素数）

rad(abc) = 2 × 59047 × 3 = 354282

log(c)/log(rad(abc)) = log(59049)/log(354282)
                     = 10·log(3)/log(354282)
                     ≈ 10.99/12.78
                     ≈ 0.86

(1+ε) = 1 で十分! ABC の不等式 log(c) ≤ (1+ε)·log(rad(abc)) は
ε = 0 でも成り立つ（この例では）
```

「質の良いABCトリプル」= ε が小さくないと成り立たない例はごく稀。
IUT理論は「どんなに質の良いトリプルでも、ε > 0 さえあれば不等式が成り立つ」と主張。

## なぜ不等式が非自明なのか（争点を数値で見る）

### 望月の主張

```
歪みの大きさ:  Σj² = 55 → 正規化後 ≈ l/6 → 大きい（l に比例）

不確定性のコスト:
  (Ind1): プロセッション → ≈ log(l) → 小さい
  (Ind2): μ_{2l} の有限群 → 定数 → 小さい
  (Ind3): 対数殻の余白 → log(p) の和 → 判別式に吸収

結論: 歪み > 不確定性のコスト → 非自明な不等式
```

### Scholze の批判（数値的に言えば）

```
Scholze: 「Ind1 が自由に作用すると、不確定性が歪みと同じオーダーになり、
          不等式は 0 ≤ 0 のような自明なものになる」

数値的には: 「Ind1 のコストが ≈ l/6 のオーダーなら、歪み l/6 を
            打ち消してしまう」
```

### 争点の核心

```
望月: 「対数殻が Ind1 を制限する。コストは log(l) 程度」
Scholze: 「対数殻による制限は不十分。コストは l/6 のオーダー」
Joshi: 「望月の群論的手法では不十分だが、幾何的手法なら制限できるはず」
```

数値で表現すると:

```
歪み: 55 (= Σj², l=11)

望月の見積もり:   不確定性コスト ≈ log(11) + 定数 ≈ 5
  → 55 - 5 = 50 >> 0 → 非自明な不等式

Scholze の見積もり: 不確定性コスト ≈ 55（歪みと同程度）
  → 55 - 55 = 0 → 自明な不等式

差: 対数殻が Ind1 を「log(l) 程度」に制限できるか否か
```

## 全3文書のまとめ

```
数値例ビュー（文書1）:
  「足し算と掛け算の分離」を数値で追跡
  核心: テータ値列 {q^{j²}} が q に圧縮されるときの歪み = Σj²

対数テータ格子ビュー（文書2）:
  「log-リンクと対数殻」を数値で追跡
  核心: 対数殻が「不確定性を閉じ込める有限の入れ物」

最終不等式ビュー（本文書）:
  「不確定性の見積もり」を数値で追跡し、ABC予想の不等式を導出
  核心: 歪み(∼l²/6) > 不確定性のコスト(∼定数+log l) → 非自明な不等式
```

**最終メッセージ**: IUT理論は、楕円曲線の q-パラメータとテータ値の間に
「掛け算を繋ぎ、足し算を切断する」ことで歪みを生み出し、
対数殻でその歪みの不確定性を有限に制御して、
ABC予想の形の不等式を導く——ということを、この3つの文書で
p=5, q=5², l=11 という一貫した数値例を通じて追跡した。

証明が正しいかどうかは2026年時点でも未決着だが、
**理論が何をしようとしているか**は、数値を追えば明確に見える。
