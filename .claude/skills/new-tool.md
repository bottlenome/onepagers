# 新規ツールプロジェクト作成

新しいツール（ゲーム以外）プロジェクトを立ち上げるスキル。

## フェイズ1: 要件ヒアリング

ユーザーと対話しながら以下を具体化する。すべて確定するまで実装に進まないこと。

### 必須ヒアリング項目

1. **ツールの目的**: 何を解決するツールか？（例: 変換、計算、可視化、管理）
2. **入力と出力**: ユーザーは何を入力し、何が得られるか？
3. **操作フロー**: 基本的な使い方の流れ
4. **プロジェクト名**: ディレクトリ名（英語kebab-case）と表示名（日本語OK）

### 任意ヒアリング項目（必要に応じて確認）

- データの保存は必要か？（localStorage等）
- インポート/エクスポート機能は必要か？
- 設定やカスタマイズ項目はあるか？
- 参考にしたいツールやイメージはあるか？

### ヒアリングの進め方

- ユーザーの最初の発言から読み取れる情報はそのまま採用し、不足分だけ質問する
- 一度に大量の質問を投げず、2〜3問ずつ聞く
- 曖昧な回答には具体的な選択肢を提示して確認する
- 全項目が確定したら、最終確認として仕様サマリーを提示し、OKをもらってからフェイズ2へ進む

## フェイズ2: 実装

1. **ディレクトリ作成**: `<project名>/` と `<project名>/docs/tasks/` を作成
2. **ツール本体作成**: `<project名>/index.html` を1ファイルで完結するツールとして作成
   - 共通スタイル: `background: #0f172a; color: #e2e8f0; font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif;`
   - 戻るリンク: `<a class="back" href="../index.html">&larr; OnePagers</a>`
   - タイトルの `<title>` は `プロジェクト名 - OnePagers` 形式
   - 色覚バリアフリー: 色の区別が必要な場合はOkabe-Itoパレットを使用し、模様でも識別可能にする
3. **プロジェクトdashboard作成**: `<project名>/dashboard.md` を以下のテンプレートで作成

```markdown
# プロジェクト名

一行説明。

## 概要

- 種別: tool
- 状態: done
- ファイル: `index.html`

## 機能

- [x] 実装済み機能1
- [x] 実装済み機能2
- [ ] 未実装機能
```

4. **ルート `index.html` 更新**: `<!-- 新しいプロジェクトはここに追加 -->` の直前にカードを追加

```html
    <a class="card" href="<project名>/index.html">
      <span class="tag">tool</span>
      <h2>表示名</h2>
      <p>一行説明。</p>
    </a>
```

5. **ルート `dashboard.md` 更新**: プロジェクト一覧テーブルに行を追加

```
| [表示名](<project名>/index.html) | tool | done | 一行説明 |
```

6. **動作確認**: ツールが正しく動作し、ダッシュボード間の説明文が一致していることを確認
