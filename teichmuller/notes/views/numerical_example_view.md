# 具体的な数値で理解する「足し算と掛け算の分離」

## この文書の目的

IUT理論の核心操作——「掛け算の次元だけを動かし、足し算の次元は固定する」——を
**実際の数値**で見せる。抽象的な定義ではなく、手で計算できるレベルに落とす。

---

## 準備: p進の世界での「2つの次元」を数値で見る

### 5進数の世界 (p=5)

5進整数とは 5 を「小さい数」と見なす世界。

```
通常の感覚: 125 は「大きい」、1/5 は「小さい」
5進の感覚: 125 = 5³ は「超小さい」、1/5 は「大きい」
```

5進付値 v₅(x) = 「x を何回 5 で割れるか」:

```
v₅(1)   = 0    （5で割れない）
v₅(5)   = 1    （1回割れる）
v₅(25)  = 2    （2回割れる）
v₅(125) = 3    （3回割れる）
v₅(7)   = 0    （5で割れない）
v₅(50)  = 2    （50 = 2 × 5²）
```

### 2つの次元の正体

任意の 5 進数 x ≠ 0 は**一意に**次のように分解できる:

```
x = 5^n × u     （n = v₅(x), u は5進単数）
    ~~~~   ~~~
    値群   単数群
    (掛け算の次元)  (足し算の次元)
```

| 数 x | 値群の成分 5^n | 単数群の成分 u | 分解 |
|------|--------------|--------------|------|
| 7 | 5⁰ = 1 | 7 | 7 = 1 × 7 |
| 25 | 5² | 1 | 25 = 5² × 1 |
| 50 | 5² | 2 | 50 = 5² × 2 |
| 35 | 5¹ | 7 | 35 = 5¹ × 7 |
| 375 | 5³ | 3 | 375 = 5³ × 3 |

**値群 = 「5を何回含むか」= 掛け算的な情報**
**単数群 = 「5を全部取り除いた残り」= 足し算に関わる情報**

### なぜ「単数群 = 足し算的」なのか

直感的でないので具体例で示す。

```
7 + 18 = 25

左辺: 7 = 5⁰ × 7,  18 = 5⁰ × 18    （どちらも値群=0、つまり単数）
右辺: 25 = 5² × 1                     （値群=2、単数=1）
```

**2つの単数（7と18）を足しただけなのに、値群が0から2に跳ね上がった！**

これが足し算と掛け算の「絡み合い」の最も単純な例:
- 掛け算的には 7 も 18 も「5とは無関係」（値群=0）
- でも足し算すると 7+18=25=5² で「5が2つ出てくる」
- **足し算が掛け算の構造を変えてしまう**

逆もある:
```
3 + 122 = 125 = 5³    （単数同士の和が値群=3を生み出す）
2 + 123 = 125 = 5³    （同上）
4 + 121 = 125 = 5³    （同上）
```

**環構造（足し算+掛け算が共存）がある限り、この「絡み合い」は避けられない。
これがABC予想を直接証明できない根本的な理由。**

## 楕円曲線からテート曲線へ（ここを丁寧に）

### ステップ1: 楕円曲線とは（復習）

楕円曲線は y² = x³ + ax + b の形の曲線。
ただし IUT理論で重要なのは「図形としての形」ではなく、以下の2つの性質:

1. **群構造**: 曲線上の点同士に「足し算」が定義できる
2. **数体上の振る舞い**: 各素数 p で「mod p に還元」したときの性質

### ステップ2: 「悪い還元」とは何か（数値例）

楕円曲線 E: y² = x³ - x を考える。

この曲線を各素数で「mod p に還元」（= 係数を mod p で見る）すると:

```
mod 3: y² = x³ - x = x(x-1)(x+1)   → きれいな曲線 (良い還元)
mod 7: y² = x³ - x = x(x-1)(x+1)   → きれいな曲線 (良い還元)
mod 2: y² = x³ - x = x(x-1)(x+1) = x·x·(x+1) = x²(x+1)
       → 重根が出る！ 曲線が「潰れる」(悪い還元)
```

**悪い還元** = mod p で曲線の形が壊れること。
特に **乗法的悪い還元** = 曲線が「結節点（ノード）」を持つ形に退化すること。

### ステップ3: 乗法的悪い還元 → テート曲線

乗法的悪い還元を持つ素数 p において、楕円曲線は p 進的に
**テート曲線**という特別な形で記述できる。

#### テート曲線を日常の言葉で

複素数の世界で、楕円曲線は「ドーナツ」（トーラス）として理解できる:

```
複素平面 C を格子 Λ = {m + nτ : m,n ∈ Z} で割ったもの:
E(C) ≅ C / Λ
```

これは「平面を長方形に切って、上下と左右を貼り合わせるとドーナツになる」というやつ。

**テート曲線は、この「p進版」**:

```
q = e^{2πiτ} とおくと、上の C/Λ は次と同型:
E(C) ≅ C× / q^Z

ここで q^Z = {..., q^{-2}, q^{-1}, 1, q, q², ...}
```

つまり: **C×（0を除いた複素数の掛け算群）を、q の冪乗で「巻き取る」とドーナツになる。**

p進世界でもまったく同じことが起きる:

```
テート曲線: E(Q_p) ≅ Q_p× / q^Z
```

Q_p×（0を除いたp進数の掛け算群）を q^Z で割ったもの。

#### 具体例: p=5, q=5²

```
Q₅× / (5²)^Z = Q₅× / {... , 5⁻⁴, 5⁻², 1, 5², 5⁴, 5⁶, ...}
```

Q₅× の元 x に対して、x と x·5² と x·5⁴ と ... は「同じ点」とみなす。
これがテート曲線。

### ステップ4: q-パラメータの意味

**q-パラメータ** = テート曲線を定義する値。楕円曲線の「p進的な形」を決める数。

- |q|_p < 1 が必要（5進では v₅(q) > 0）
- q が p に「近い」ほど、楕円曲線の退化が「ひどい」
- v₅(q) = 楕円曲線が「どれだけ潰れているか」の度合い

**q-パラメータの対数 log|q| が、楕円曲線の「高さ」に直結する。**
これがABC予想の左辺に対応するので、q は理論全体の核心的な対象。

```
ABC予想の不等式:  (1/6)·log|q| ≤ (1+ε)·(log-diff + log-cond)
                   ~~~~~~~~~~
                   ここが q-パラメータ
```

### ステップ5: テート曲線上のl-等分点

楕円曲線には「l-等分点」がある: l回足すと原点に戻る点。

テート曲線 E ≅ Q_p× / q^Z 上では、l-等分点は具体的に書ける:

```
E[l] = { ζ_l^a · q^{b/l} : a,b ∈ Z/lZ }
```

ここで ζ_l = e^{2πi/l} はlの原始根。

つまり l-等分点は:
- **ζ_l の冪** = 1の l乗根 → 「回転」の成分（乗法的部分空間 μ_l）
- **q^{1/l}** = q の l乗根 → 「巻き取り」の成分

この2つの成分が、完全系列:

```
0 → μ_l → E[l] → Z/lZ → 0
```

に対応する。**μ_l が「乗法的部分空間」（global multiplicative subspace）。**

### ステップ6: テータ関数 — 「何をどう測るか」

#### テータ関数とは何か

テート曲線上のテータ関数は、曲線上の「特別な関数」。
テート曲線の明示的な級数表示（[EtTh, Prop 1.4]）:

```
Θ(u) = Σ_{n∈Z} (-1)^n · q^{(n+1/2)²/2} · u^{n+1/2}

     = ... + q^{9/8}·u^{-3/2} - q^{1/8}·u^{-1/2} + q^{1/8}·u^{1/2} - q^{9/8}·u^{3/2} + ...
```

ここで u は Q_p× / q^Z の座標。

#### なぜ j² が出てくるのか

テータ関数を l-等分点 u = q^{j/l}（j = 1, ..., l*）で評価すると:

```
Θ(q^{j/l}) の主要項 ∼ q^{j²/2l} の定数倍
```

したがって、Θ の適切な冪（l乗根の逆数を取るなど）を考えると:

```
テータ値 ∼ q^{j²}    （主要項として）
```

**j² が出てくるのは、テータ関数の級数展開で指数が二次形式 (n+1/2)² だから。**
これはガウス分布 e^{-x²} の離散版: n² の形が自然に現れる。

### ステップ7: 数値例（p=5, q=5², l=11, l*=5）

```
テート曲線: E ≅ Q₅× / (5²)^Z
q = 5²,  v₅(q) = 2
```

各 l-等分点 j = 1,...,5 でのテータ値（主要項）:

| j | j² | テータ値 ∼ q^{j²} = 5^{2j²} | v₅(テータ値) = 2j² |
|---|----|----|------|
| 1 | 1 | 5² = 25 | 2 |
| 2 | 4 | 5⁸ = 390,625 | 8 |
| 3 | 9 | 5¹⁸ ≈ 3.8 × 10¹² | 18 |
| 4 | 16 | 5³² ≈ 2.3 × 10²² | 32 |
| 5 | 25 | 5⁵⁰ ≈ 8.9 × 10³⁴ | 50 |

テータ値の付値の列: **2, 8, 18, 32, 50**

q-パラメータ自体の付値: **2**

### ここまでの流れを一気通貫で

```
楕円曲線 E/Q
  ↓ 素数 p=5 で乗法的悪い還元
テート曲線 E ≅ Q₅×/q^Z  (q=5²)
  ↓ l=11 の l-等分点で
テータ関数を評価
  ↓ 主要項を取ると
テータ値列 {q^{j²}} = {5², 5⁸, 5¹⁸, 5³², 5⁵⁰}
  ↓ 付値を見ると
{2, 8, 18, 32, 50}  ← これが Θ-リンクで「向こう側」に送られるデータ
```

## Θ-リンクが「やること」を数値で見る

### まず「2つの世界」とは何か — なぜ必要なのか

普通の数学では、楕円曲線の q-パラメータを直接いじって不等式を証明しようとする。
しかし q-パラメータは環構造（足し算と掛け算の両方）に縛られていて、
自由にいじれない。

望月の発想: **同じ楕円曲線のコピーを2つ用意して、片方の「テータ値」を
もう片方の「q-パラメータ」と対応づける。**

これが「2つの世界」。

#### たとえ話: Excelシートのコピー

```
┌─ シートA ──────────────────────────────┐
│                                        │
│  セルA1: 25      ← q の値              │
│  セルA2: =A1^4   ← テータ値（自動計算）  │
│  セルA3: =A1+A2  ← 足し算（数式で連動）  │
│                                        │
│  全セルが数式でガチガチに連動している     │
└────────────────────────────────────────┘
```

シートAの中では、q を変えるとテータ値も足し算の結果も全部連動して変わる。
だから q とテータ値の間に不等式を作ることができない（同じシート内で
「左辺を大きくしたら右辺も大きくなる」ので差がつかない）。

```
┌─ シートA ──────────────┐     ┌─ シートB ──────────────┐
│                        │     │                        │
│  q_A = 25              │     │  q_B = ???              │
│  テータ値: 25,390625,...│     │  テータ値: 計算できる   │
│  足し算: 連動           │     │  足し算: 独自に連動     │
│                        │     │                        │
└──────────┬─────────────┘     └──────────┬─────────────┘
           │                              ▲
           │   「シートAのテータ値」を      │
           │    コピーして                  │
           └──「シートBの q_B」に ─────────┘
               ペーストする
               （ただし「値のみ貼り付け」！
                 数式は持ち込めない！）
```

**「繋がる」= 値のみ貼り付け（ペースト）ができること。**

- **繋がるもの**: 掛け算の情報（「5の何乗か」= 付値）だけはコピーできる
- **繋がらないもの**: 足し算の情報（数式リンク）は持ち込めない

「値のみ貼り付け」だから、シートBではシートAの数式（足し算の関係）が
切れている。これがIUT理論の核心: **掛け算だけ繋いで、足し算を切断する。**

### 数値で見る「2つの世界」

```
┌──── 世界A ────────────────────────────────────────────┐
│                                                       │
│  q_A = 5²     (v₅ = 2)                               │
│  テータ値: 5², 5⁸, 5¹⁸, 5³², 5⁵⁰                     │
│  足し算の例: 5² + 5⁸ = 25 + 390625 = 390650           │
│             = 2 × 5² × 7813                          │
│  （q_A とテータ値と足し算が全部「同じ環」で連動）       │
│                                                       │
└───────────────────────┬───────────────────────────────┘
                        │
                        │  Θ-リンク（値のみ貼り付け）
                        │
                        │  送れるもの: 「5の何乗か」（付値）
                        │  送れないもの: 足し算の結果、単数群
                        │
                        ▼
┌──── 世界B ────────────────────────────────────────────┐
│                                                       │
│  世界Bは世界Aの「コピー」だが、独立した環構造を持つ      │
│                                                       │
│  q_B: Θ-リンクにより、世界Aのテータ値列の付値           │
│       {2, 8, 18, 32, 50} を受け取る                    │
│                                                       │
│  世界Bの足し算: 世界B独自のもの                         │
│  世界Aで成り立った「25 + 390625 = 390650」は            │
│  世界Bでは意味を持たない（数式リンクが切れているから）    │
│                                                       │
└───────────────────────────────────────────────────────┘
```

### 「q_B = ???」の意味 — 「決められない」ではなく「歪んで届く」

ここが混乱しやすいポイント。

世界Bにも q_B は**ちゃんと存在する**（世界Bは楕円曲線のコピーだから）。
世界B単独で見れば v₅(q_B) = 2（元の楕円曲線と同じ）。

**??? の意味は「世界Aから見たとき、q_B がいくつなのか特定できない」ということ。**

なぜか？ Θ-リンクは以下の対応だけを作るから:

```
世界Aのテータ値 {q_A^{j²}}  ←→  世界Bの q_B
```

世界Aから世界Bに「届く」情報は、テータ値列の掛け算的情報（付値）だけ。
足し算の情報が切れているので、世界Aの座標系で q_B の値を
正確に書き下すことができない。

#### 数値で見ると

```
世界A側から見える情報:
  テータ値列の付値 = {2, 8, 18, 32, 50}
  ↓ Θ-リンクで世界Bに送られる

世界B側に届く情報:
  q_B の付値 = 2  （世界B内部ではこれは確定している）

しかし世界A側からは:
  「q_B = 5² × (何か)」の「何か」の部分（= 単数部分）が見えない
  なぜなら単数部分は足し算の情報に依存し、足し算は切断されているから

  q_B = 5² × u_B   （u_B は世界Bの単数。世界Aからは見えない。）
       ~~~   ~~~
       見える  見えない（これが ???）
```

**つまり「???」= 掛け算の骨格（付値=2）は分かるが、肉付け（単数部分）が不明。**

### 何が「動いた」か、何が「固定」か

**掛け算の次元（値群）: Θ-リンクで繋がった → 「歪み」が生じた**

```
世界A → 世界B に送られるとき:

  世界Aでの付値:  q_A は 2、テータ値列は {2, 8, 18, 32, 50}
  世界Bでの付値:  q_B は 2

  Θ-リンクの対応: 世界Aのテータ値列 {2,8,18,32,50} 全体が
                  世界Bの q_B の付値 2 に「押し込まれる」
```

これが**値群の伸縮**: 合計110の付値が、一つの「2」に対応づけられる。
5つの異なる値 {2,8,18,32,50} が1つの「2」に圧縮される歪みが
最終的に不等式の源になる。

**足し算の次元（単数群）: 切断されている**

```
世界A: 7 + 18 = 25 が成り立つ（5進で v₅ が 0→2 に跳ぶ）

世界B: この「7 + 18 = 25」は世界Bでは意味をなさない。
       世界Bの「7」と世界Aの「7」は、Θ-リンクで繋がっていない。
       世界Bには世界B独自の足し算がある。
       （Excelで「値のみ貼り付け」したら数式が消えるのと同じ）
```

## 「片方だけ動かす」を一枚の表で

q = 5², l* = 5 の場合:

```
        ┃  値群 (掛け算の次元)     ┃  単数群 (足し算の次元)
━━━━━━━━╋━━━━━━━━━━━━━━━━━━━━━━━╋━━━━━━━━━━━━━━━━━━━━━━━
世界A    ┃  v₅(q_A) = 2           ┃  世界Aの単数群 U_A
        ┃  テータ値列の付値:       ┃   （U_A 内で足し算ができる）
        ┃  2, 8, 18, 32, 50      ┃
━━━━━━━━╋━━━━━━━━━━━━━━━━━━━━━━━╋━━━━━━━━━━━━━━━━━━━━━━━
Θ-リンク ┃  ← 値のみ貼り付け →    ┃  ← 数式リンク切断 →
        ┃  {2,8,18,32,50} → 2   ┃  U_A と U_B は無関係
━━━━━━━━╋━━━━━━━━━━━━━━━━━━━━━━━╋━━━━━━━━━━━━━━━━━━━━━━━
世界B    ┃  v₅(q_B) = 2           ┃  世界Bの単数群 U_B
        ┃  （受け取り側）          ┃   （U_B 内で独自の足し算）
━━━━━━━━╋━━━━━━━━━━━━━━━━━━━━━━━╋━━━━━━━━━━━━━━━━━━━━━━━

       「繋がる」                 「切断される」
    （値のみコピー）             （数式は消える）
```

**通常の環構造では**:
- 値群と単数群は分配法則で結合している
- 例: 5² × 7 + 5² × 18 = 5² × (7+18) = 5² × 25 = 5⁴
- 値群の情報（5²）と単数群の情報（7, 18）が混ざり合う

**Θ-リンクでは**:
- 値群の情報だけを「値のみ貼り付け」で別の世界に送る
- 単数群の情報は送れない → 分配法則が成り立たない → 環構造が壊れる
- **環構造が壊れるからこそ、2つの世界の間に「歪み」＝不等式が生じる**

## 「歪み」が不等式を生むメカニズム（数値で）

### 対数体積で見る

「対数体積」= 付値の（正規化された）合計、と思ってください。

**世界Aのテータ値列の対数体積**:

```
Σ v₅(q^{j²}) = v₅(q) × Σ j²
             = 2 × (1 + 4 + 9 + 16 + 25)
             = 2 × 55
             = 110
```

**世界Bの q-パラメータの対数体積**:

```
v₅(q_B) = 2
```

### なぜこの2つの数が違うのか — ここが核心

「付値で繋がっているなら対数体積も同じでは？」という疑問はもっともです。
ポイントは **Θ-リンクが「等号」ではなく「同一視」である** ことです。

```
Θ-リンクの対応:

  世界Aのテータ値列  {q_A^1, q_A^4, q_A^9, q_A^16, q_A^25}
        ‖ Θ-リンク（この等号が「歪んでいる」）
  世界Bの q_B

  つまり「5個の値の列」全体が「1個の値」に対応する
```

これは「等しい」のではなく**「無理やり同一視する」**操作です。

#### たとえ話

5人の社員の給料明細（25万、39万、...）を1枚の封筒に詰めて送ったとする。
届いた封筒は1つ。中身は確かに5人分のデータだが、
届いた側（世界B）では「封筒1つ分 = q_B 1個分」として扱うしかない。

```
世界A: 5人分の給料を個別に計算 → 合計110万円
世界B: 届いた封筒1つ → 「q_B = 2万円」として処理

世界Aから見ると: 110万円分の情報を送ったはず
世界Bから見ると: 2万円分しか届いていない

この差 = 110 vs 2 が「歪み」
```

### 歪みの比率

```
世界Aの対数体積:  110
世界Bの対数体積:    2
               ─────
歪み率:        110 / 2 = 55 = Σ j²
```

**「2で割っている」のではなく、2つの世界の対数体積を比較（割り算）しているだけ。**

- 分子の110: テータ値列の付値の合計（5個の値を全部足した）
- 分母の2: q_B の付値（1個の値）
- この比 55 = Σj² が、Θ-リンクが生んだ「歪み」の大きさ

付値で繋がっているのに対数体積が違うのは、
**「5個の列」と「1個の値」を同一視するという非自明な操作**をしているから。
等号ではなく圧縮。だから情報量に差が出る。

### これがABC予想にどう繋がるか

一般の l（l* = (l-1)/2）では:

```
歪み率 = Σ_{j=1}^{l*} j² ≈ l³/24
```

この歪み率を l で割って正規化すると:

```
(1/l) × Σ j² ≈ l²/24
```

さらに、テータ値列は l* 個の成分からなるので、1成分あたりの平均付値は:

```
(Σ j²) / l* ≈ l²/6
```

一方、q-パラメータの対数は楕円曲線の高さ ht_E に対応する。

IUT IVで l を ht_E と同じオーダーに選ぶと（l ∼ ht_E）、上の見積もりから:

```
(1/6) × ht_E ≤ (1+ε) × (log-diff + log-cond) + 定数
```

の形の不等式が導かれる。
左辺は (1/6) × log(q) に対応し、右辺は判別式と導手に対応する。

### 具体的な数字の流れ

q = 5², l = 11 (l* = 5) の場合:

```
1. テータ値列: {5², 5⁸, 5¹⁸, 5³², 5⁵⁰}
   付値列: {2, 8, 18, 32, 50}

2. 対数体積（合計）: 110

3. q-パラメータの対数体積: 2

4. 比率: 110/2 = 55

5. 1/l で正規化: 55/11 = 5 = l*

6. 1/l* で平均: 55/5 = 11 = l

7. この「11」が最終的に (1/6)·ht_E の左辺になる
   （厳密にはもう少し複雑だが、最高次項として）
```

## なぜ「分離」が不等式を生むのか（核心の直感）

通常の環構造の中では:

```
足し算と掛け算が絡み合っている
  → 値群の変化（q → q^{j²}）を見積もろうとすると、
    足し算由来の項（ガウス極）が爆発する
  → 不等式が vacuous（空虚）になる（何も言えない）
```

IUT理論では:

```
足し算の次元と掛け算の次元を分離する
  → 掛け算の次元（値群）だけを q → q^{j²} で変形
  → 足し算の次元は「対数殻」の中に閉じ込められる
  → 対数殻のサイズは有限（明示的に計算可能）
  → 歪みの量 = Σj² / q の比率が有限に制御される
  → 非自明な不等式が得られる
```

**分離しないと**: ガウス極のせいで「∞ ≤ 何か」になり、何も言えない。
**分離すると**: ガウス極が消えて「有限の量 ≤ 何か」になり、ABC予想の形になる。

## 古典的タイヒミューラーとの数値的対応

### 古典: z → ζ = Kx + iy

```
リーマン面のタイヒミューラー変形:

z = x + iy = (3, 4)    （2つの実次元）

変形後: ζ = 2x + iy = (6, 4)     （K=2 で x方向だけ2倍）

動いた: x成分  3 → 6 (×2)
固定:   y成分  4 → 4 (そのまま)

歪みの量 = K = 2（擬等角定数）
```

### IUT: 値群を j² 倍に「伸縮」

```
数体のIUT変形:

q = 5²   （v₅ = 2, 単数 = 1 の2つの次元）

変形後 (j=3): q^{j²} = q^9 = 5¹⁸

動いた: 値群  2 → 18 (×9)
固定:   単数群は別世界に属するので比較不能（= 固定というより切断）

歪みの量 = Σj² = 55（l*=5の場合）
```

| | 古典 | IUT |
|---|---|---|
| 対象 | z = x + iy | q = p^n × u |
| 2つの次元 | 実部 x / 虚部 y | 値群 n / 単数群 u |
| 変形 | x → Kx | n → j²n |
| 固定 | y → y | u → （別世界、繋がない）|
| 歪みの量 | K（擬等角定数）| Σj²（テータ値の合計）|
| 結論 | テータ距離の上界 | ABC不等式 |
